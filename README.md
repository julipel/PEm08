## 📖 Описание

**Мониторинг конкурентов** — это AI-ассистент для комплексного анализа конкурентной среды. Приложение использует OpenAI GPT модели для анализа текстов, изображений и веб-сайтов конкурентов, предоставляя структурированные инсайты, рекомендации и оценки.

### Ключевые особенности:

- 🎯 **Мультимодальный анализ** — работа с текстом, изображениями и веб-сайтами
- 🤖 **AI-анализ** — использование OpenAI GPT-4o-mini для глубокого анализа
- 📊 **Структурированные результаты** — оценки, инсайты и практические рекомендации
- 📋 **История запросов** — автоматическое сохранение последних 10 анализов
- 🌐 **Веб-интерфейс** — современный и удобный UI
- 🖥️ **Desktop приложение** — нативное Windows приложение на PyQt6
- 📡 **REST API** — полнофункциональное API для интеграций

## 🚀 Установка

### Требования

- **Python 3.9+** (рекомендуется 3.10+)
- **OpenAI API ключ** (получить: https://platform.openai.com/api-keys)
- **Chrome/Chromium** (для парсинга сайтов, устанавливается автоматически)
- **Интернет-соединение** (для работы с OpenAI API и парсингом)

### Шаг 1: Клонирование репозитория

```bash
git clone https://github.com/Toxap/pem08.git
cd pem08
```

### Шаг 2: Создание виртуального окружения

**Windows:**
```powershell
python -m venv venv
venv\Scripts\Activate.ps1
```

**Linux/Mac:**
```bash
python -m venv venv
source venv/bin/activate
```

### Шаг 3: Установка зависимостей

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

## ⚙️ Настройка

### Переменные окружения

1. Скопируйте файл примера:
```bash
cp env.example.txt .env
```

2. Откройте `.env` и настройте параметры:

```env
# OpenAI API (ОБЯЗАТЕЛЬНО)
OPENAI_KEY=your_openai_api_key_here

# Модели (опционально)
OPENAI_MODEL=gpt-4o-mini
OPENAI_VISION_MODEL=gpt-4o-mini

# API Configuration
API_HOST=0.0.0.0
API_PORT=8000

# Competitor URLs (опционально, через запятую)
COMPETITOR_URLS=example.com,test.com
```

### Получение OpenAI API ключа

1. Перейдите на https://platform.openai.com/api-keys
2. Войдите в аккаунт (или создайте новый)
3. Нажмите "Create new secret key"
4. Скопируйте ключ и вставьте в `.env`

**Важно:** Никогда не публикуйте API ключ в открытом доступе!

### Доступные модели OpenAI

- `gpt-4o` — лучшее качество, дороже
- `gpt-4o-mini` — баланс цена/качество (рекомендуется)
- `gpt-4-turbo` — альтернатива GPT-4o
- `gpt-3.5-turbo` — быстрее и дешевле, но менее качественно

## ▶️ Запуск

### Backend сервер

**Способ 1: Через run.py (рекомендуется)**
```bash
python run.py
```

**Способ 2: Через uvicorn напрямую**
```bash
python -m uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```

Сервер запустится и будет доступен по адресу: **http://localhost:8000**

### Веб-интерфейс

После запуска backend сервера откройте в браузере:
- **Веб-интерфейс:** http://localhost:8000
- **API документация (Swagger):** http://localhost:8000/docs
- **ReDoc документация:** http://localhost:8000/redoc

### Desktop приложение

**Требование:** Backend сервер должен быть запущен!

**Способ 1: Запуск исходного кода**
```bash
cd desktop
pip install -r requirements.txt
python main.py
```

**Способ 2: Запуск собранного .exe**
```bash
# Сначала соберите .exe (один раз)
cd desktop
python build.py

# Затем запустите
.\dist\CompetitorMonitor.exe
```

## 📚 API Документация

### Базовый URL

```
http://localhost:8000
```

### Эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/` | Главная страница (веб-интерфейс) |
| POST | `/analyze_text` | Анализ текста конкурента |
| POST | `/analyze_image` | Анализ изображения конкурента |
| POST | `/parse_demo` | Парсинг и анализ сайта по URL |
| GET | `/history` | Получение истории запросов |
| DELETE | `/history` | Очистка истории запросов |
| GET | `/health` | Проверка работоспособности |
| GET | `/docs` | Swagger UI документация |
| GET | `/redoc` | ReDoc документация |

### Примеры запросов

#### Анализ текста

```bash
curl -X POST "http://localhost:8000/analyze_text" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Наша компания предлагает уникальные решения для бизнеса..."
  }'
```

#### Анализ изображения

```bash
curl -X POST "http://localhost:8000/analyze_image" \
  -F "file=@banner.jpg"
```

#### Парсинг сайта

```bash
curl -X POST "http://localhost:8000/parse_demo" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "example.com"
  }'
```

Подробная документация API доступна в файле [docs.md](docs.md) и в интерактивной документации: http://localhost:8000/docs

## 🔧 Решение проблем

### Сервер не запускается

**Проблема:** `ModuleNotFoundError` или ошибки импорта

**Решение:**
```bash
# Убедитесь, что виртуальное окружение активировано
# Windows:
venv\Scripts\Activate.ps1

# Проверьте установку зависимостей
pip install -r requirements.txt
```

### API ключ не работает

**Проблема:** Ошибки при запросах к OpenAI

**Решение:**
1. Проверьте, что ключ в `.env` файле указан правильно
2. Убедитесь, что ключ начинается с `sk-`
3. Проверьте баланс на https://platform.openai.com/account/usage
4. Убедитесь, что модель доступна для вашего аккаунта

### Парсинг сайтов не работает

**Проблема:** Ошибки при парсинге сайтов

**Решение:**
1. Убедитесь, что установлен Chrome/Chromium
2. WebDriver Manager установит драйвер автоматически
3. Проверьте доступность сайта в браузере
4. Некоторые сайты могут блокировать автоматический доступ

### Desktop приложение не подключается

**Проблема:** "Сервер недоступен"

**Решение:**
1. Убедитесь, что backend сервер запущен
2. Проверьте, что сервер доступен на `http://localhost:8000`
3. Проверьте firewall настройки
4. В `api_client.py` можно изменить `base_url` при необходимости

### Порты заняты

**Проблема:** `Address already in use`

**Решение:**
1. Измените порт в `.env`: `API_PORT=8001`
2. Или завершите процесс, использующий порт 8000

## 🛡️ Безопасность

⚠️ **Важные рекомендации:**

1. **API ключи** — никогда не публикуйте `.env` файл в репозиторий
2. **Git** — убедитесь, что `.env` в `.gitignore`
3. **HTTPS** — в продакшене обязательно используйте HTTPS
4. **CORS** — настройте CORS для конкретных доменов (сейчас `allow_origins=["*"]`)
5. **Валидация** — все входные данные валидируются через Pydantic
6. **Лимиты** — рассмотрите добавление rate limiting для API
